# Pull Requests

Esta sección contiene la documentación detallada de todos los Pull Requests del proyecto.

## Cómo se Genera

Cada vez que se abre o actualiza un Pull Request, el sistema automáticamente:

1. Analiza los cambios realizados
2. Detecta módulos afectados
3. Identifica el tipo de cambio (feat, fix, breaking)
4. Genera un documento completo con toda la información relevante
5. Lo almacena en esta sección

## Estructura de Documentación

Cada PR incluye:

- **Información General**: Título, autor, fecha, tipo
- **Módulos Afectados**: Componentes del sistema modificados
- **Descripción**: Detalle funcional del cambio
- **Archivos Modificados**: Lista completa de archivos
- **Estadísticas**: Commits, líneas añadidas/eliminadas
- **Impacto Funcional**: Clasificación del cambio
- **Checklist de Revisión**: Estado de revisión
